# Online Shop API — презентація проєкту

English version: [README.md](README.md)

## Що це за проєкт

`online_shop` — це backend-проєкт для інтернет-магазину, створений на FastAPI. Він демонструє готову основу для e-commerce системи: каталог товарів, категорії, клієнтів, замовлення, авторизацію, ролі користувачів, адміністративну панель, документацію API та автоматизовані тести.

Проєкт підходить як портфоліо-рішення, демо для клієнта або стартова технічна база для реального онлайн-магазину.

## Backend для онлайн-магазину

Серверна частина побудована на FastAPI та надає REST API для основних процесів магазину:

- реєстрація та вхід користувачів;
- керування товарами та категоріями;
- створення й перегляд замовлень;
- контроль залишків на складі;
- розділення доступу між адміністратором і покупцем;
- перевірка стану застосунку через health endpoints.

## Українська адміністративна панель

Проєкт має HTML admin panel українською мовою. Адміністратор може працювати з магазином через браузер без прямого використання API.

Основні можливості адмін-панелі:

- **Dashboard** — огляд ключових показників магазину: товари, категорії, клієнти, замовлення та статуси замовлень.
- **Products** — перегляд, пошук, створення, редагування та безпечне видалення товарів.
- **Categories** — керування категоріями товарів, пошук, створення, редагування та видалення невикористаних категорій.
- **Orders** — перегляд замовлень, фільтрація за статусами та зміна статусів замовлень.
- **Customers** — перегляд клієнтів, пошук і сторінки деталей клієнтів із пов’язаними замовленнями.
- **Low stock** — окрема сторінка для товарів із низькими залишками.

## Swagger API документація

API має інтерактивну документацію Swagger UI. Через Swagger можна переглядати endpoints, авторизуватися, тестувати запити та демонструвати роботу backend без окремого frontend-застосунку.

Демо Swagger UI:

```text
https://online-shop-api-z9y4.onrender.com/docs
```

## База даних і Render

Для production deployment проєкт використовує PostgreSQL на Render. Застосунок читає підключення до бази даних через `DATABASE_URL`, а міграції структури бази виконуються через Alembic.

Локально проєкт може працювати з SQLite для зручної розробки, але production-оточення орієнтоване на PostgreSQL.

## Авторизація та ролі

У системі є дві основні ролі:

- **admin** — керує товарами, категоріями, замовленнями, клієнтами та переглядає адміністративну панель;
- **customer** — створює профіль клієнта, переглядає каталог і оформлює власні замовлення.

API використовує JWT authentication. Для HTML admin panel використовується окрема сесія адміністратора через захищений cookie-based login.

## Автоматизовані тести

Проєкт має автоматизовані тести, які перевіряють API, авторизацію, ролі, товари, категорії, клієнтів, замовлення, admin UI, міграції та допоміжні scripts.

Основна команда для запуску тестів:

```bash
pytest -q
```

Тести також використовуються у CI, щоб перевіряти стабільність проєкту перед оновленнями.

Типовий демо-сценарій:

1. Адміністратор входить в admin panel.
2. Адміністратор створює категорії та товари.
3. Покупець реєструється через API.
4. Покупець створює customer profile.
5. Покупець переглядає каталог і створює замовлення.
6. Адміністратор переглядає замовлення та змінює його статус.
7. Система оновлює залишки товарів і показує товари з низьким stock.

## Що можна адаптувати для реального клієнта

Для реального бізнесу проєкт можна розширити або кастомізувати:

- дизайн і тексти адміністративної панелі під бренд клієнта;
- структуру каталогу, додаткові поля товарів і варіації товарів;
- статуси замовлень і бізнес-логіку обробки замовлень;
- інтеграцію з платіжними системами;
- інтеграцію з доставкою або CRM;
- email/SMS/Viber/Telegram повідомлення для клієнтів і адміністраторів;
- імпорт та експорт товарів;
- аналітичні звіти для продажів і залишків;
- повноцінний customer-facing frontend;
- багатомовність для публічної частини магазину;
- production monitoring, backups і додаткові security checks.

## Коротко для замовника

Цей проєкт показує, що backend інтернет-магазину вже має основу для реальної роботи: API, базу даних, авторизацію, ролі, адмін-панель українською, документацію Swagger, deployment на Render із PostgreSQL та автоматизовані тести. Його можна використовувати як демонстрацію можливостей або як стартову точку для створення індивідуального онлайн-магазину.
